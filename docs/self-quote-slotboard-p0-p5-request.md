# `/self-quote` 슬롯 보드형 견적 관계도 P0 전환 계획

## Summary
`/self-quote`의 기존 노드 기반 견적 관계도와 하단 목록/장바구니 3영역을 화면에서 제거하고, 첨부 HTML의 `2a` 방향인 **CSS 추상형 PC 조립 슬롯 보드**로 전환한다. 다음 구현은 **P0까지만** 진행하고 멈춘다. P1~P5는 남겨두며, 사용자가 다음 구현을 요청하기 전에는 진행하지 않는다.

## Key Changes
- `/self-quote`는 슬롯 보드 전용 화면으로 재구성한다.
  - 기존 `PC 카테고리 / 전체 부품 목록 / 견적 장바구니` JSX는 legacy 컴포넌트로 추출해 보존하되 렌더링하지 않는다.
  - 홈/내 견적함 등 다른 `BuildDependencyGraph` 사용처는 건드리지 않는다.
  - 기존 AI 추천 조합을 셀프 견적에 적용하면 8개 부품이 슬롯에 꽂힌 상태로 보인다.

- 새 `SlotBoard` UI를 추가한다.
  - 8개 슬롯: CPU, 메인보드, RAM, GPU, SSD, 파워, 케이스, 쿨러.
  - 2a 형식의 CSS 추상형 부품 형태를 사용하고 외부 이미지/bitmap은 쓰지 않는다.
  - 슬롯 정보는 `현재 quote draft items` 기준으로 표시한다.
  - 슬롯 내부는 `카테고리 + 상품명 + 가격`을 표시한다.
  - 빈 슬롯은 점선 실루엣과 `+ 부품 선택` 문구로 표시한다.
  - RAM은 고정 시각 슬롯 4칸, SSD는 고정 시각 슬롯 2칸으로 표현한다. 실제 메인보드 슬롯 수라고 단정하지 않는다.
  - 아무 부품도 없으면 8개 빈 슬롯을 보여준다.

- 우측 후보 패널을 추가한다.
  - 데스크톱: 초기에는 닫힘, 슬롯 클릭 시 오른쪽 split 패널이 열린다.
  - 패널은 X와 ESC로 닫는다.
  - 선택 슬롯은 파란 ring으로 강조하고 패널 제목에 카테고리를 표시한다.
  - 후보는 기존 `GET /api/parts` 목록 API와 `compatibilitySource=QUOTE_DRAFT_CURRENT`를 사용한다.
  - 후보는 PASS/WARN만 보여주고 FAIL은 숨긴다. 개수는 세지 않고 `안 맞는 후보는 숨김` 정책 문구만 표시한다.
  - 후보는 20개 단위 무한스크롤로 불러온다.
  - 기본 정렬은 가격낮은순, 옵션은 가격높은순/이름순을 제공한다.
  - 후보 카드는 이미지, 상품명, 제조사, 가격, 상태 배지, 담기/교체 버튼을 표시한다.
  - WARN 후보는 즉시 적용하되 `간섭 주의` 배지를 유지한다.

- 슬롯 조작 정책을 고정한다.
  - 단일 부품 슬롯은 후보 선택 시 즉시 교체한다.
  - 단일 부품 제거는 슬롯 hover/focus 액션으로 제공한다.
  - RAM/SSD는 P0에서 패널 상단에 현재 항목 목록을 보여주고 개별 제거/교체를 제공한다.
  - RAM/SSD 후보 선택 시 어떤 항목을 바꿀지 패널에서 선택한다.
  - 비로그인 사용자가 슬롯을 클릭하면 기존 정책처럼 로그인으로 리다이렉트한다.

- 하단 상태바를 추가한다.
  - `총액`, `장착 N/8`, `내 견적함 저장`, `구매하기`를 표시한다.
  - 미완성 견적도 저장/구매 모두 허용하되, `미장착 슬롯 N개가 있습니다` 주의 배너를 명확히 보여준다.
  - 저장 성공 후에는 현재 화면에 머물고 성공 메시지와 `내 견적함 보기` 링크를 표시한다.
  - FAIL 상태의 구매 차단은 P0 범위가 아니며, P2 이후 문제 표시 단계에서 다시 다룬다.

- P0에서 제외하고 남길 작업:
  - P1: 기존 graph API를 검증 데이터로 재사용해 관계선 라벨 추가.
  - P2: WARN=`간섭 주의`, FAIL=`안 맞음`을 문제 슬롯+관계선 강조로 표시.
  - P3: 슬롯 상세/복수 부품 관리 고도화.
  - P4: 장착 flash/FLIP 등 애니메이션.
  - P5: 캡처 QA와 반응형 polish.

## Public APIs / Interfaces
- 백엔드 API, route path, auth guard, DB schema는 변경하지 않는다.
- P0 후보 패널은 기존 `GET /api/parts`를 재사용한다.
- 기존 `POST /api/build-graphs/resolve`는 P0에서는 화면 표시의 필수 의존으로 쓰지 않는다. P1부터 관계선/검증 표시용으로 재사용한다.
- 신규 프론트 내부 컴포넌트만 추가한다.
  - 예: `SlotBoard`, `SlotCandidatePanel`, `SlotStatusBar`, `LegacySelfQuoteListSections`.

## Test Plan
- 테스트를 먼저 작성/수정한다.
  - `/self-quote`에서 기존 메인 관계도 대신 슬롯 보드가 보이는지 검증.
  - 8개 슬롯과 빈 슬롯 `+ 부품 선택` 표시 검증.
  - AI 추천 조합 적용 후 각 부품이 슬롯에 꽂힌 상태로 표시되는지 검증.
  - 슬롯 클릭 시 우측 후보 패널이 열리고 선택 슬롯 ring/패널 제목이 표시되는지 검증.
  - 후보 패널이 `GET /api/parts`를 20개 단위로 호출하고 PASS/WARN 후보만 보여주는지 검증.
  - 후보 선택 시 기존 add/replace 흐름으로 슬롯과 하단 상태바가 갱신되는지 검증.
  - RAM/SSD 현재 항목 목록, 개별 제거/교체 동작 검증.
  - 하단 상태바의 총액, 장착 수, 저장/구매 버튼, 미완성 주의 배너 검증.
  - 모바일에서는 세로 슬롯 리스트와 bottom sheet 후보가 겹치지 않는지 검증.

- 실행 명령:
  - `cd apps/web && npm run test -- self-quote.spec.ts`
  - `cd apps/web && npm run test -- home.spec.ts`
  - `cd apps/web && npm run build`

- UI QA 필수:
  - 데스크톱 `/self-quote` 빈 슬롯 상태 캡처.
  - 데스크톱 AI 추천 조합 적용 후 슬롯 보드 캡처.
  - 데스크톱 우측 후보 패널 열린 상태 캡처.
  - 모바일 세로 슬롯 리스트 + bottom sheet 캡처.

## Assumptions
- 다음 구현 요청이 오면 P0만 구현하고 멈춘다.
- `/self-quote`는 슬롯 보드 전용 화면으로 본다.
- 기존 목록형 셀프 견적 UI는 삭제하지 않고 legacy 컴포넌트로 보존하지만 화면에는 노출하지 않는다.
- 메인보드의 실제 RAM/SSD 슬롯 수는 현재 데이터로 안정적으로 알 수 없으므로, P0의 RAM/SSD 미니 슬롯은 시각적 표현이다.
