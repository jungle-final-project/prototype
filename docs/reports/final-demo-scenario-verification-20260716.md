# 최종 발표 시나리오 검증 보고서

## 검증 기준

- 기준 커밋: `45c63b6` (`upstream/main`, PR #199 포함)
- 검증 일자: 2026-07-16
- 대상: 메인 AI 검색 → 최소 게이밍 견적 3안 → 목표 FPS GPU 변경 → 의도적 케이스 호환 실패 → AI 복구 → 저장 견적 비교 → 견적 상세
- AI 모델 정책: 기존 `BUILD_CHAT_54_MINI_FAST (gpt-5.4-mini)` 유지
- 원칙: 추천·교체 후보의 사실과 안전성은 DB/FPS seed/Tool이 결정하고, 일반 변경은 확인 전 quote draft를 바꾸지 않는다. 현재 케이스 `size` FAIL을 해소하는 서버 검증 단일 복구안만 명령형 요청 자체를 적용 동의로 본다.
- 후속 UI 기준: 게임 예상 성능의 최초 선택은 `배그 / 4K`다. 아래의 QHD 수치는 명시적인 QHD 목표 GPU 변경 프롬프트를 검증한 실측 기록이며 최초 화면 기본값을 뜻하지 않는다.

## 결론

최종 발표 핵심 체인은 실제 API와 실제 웹에서 통과했다. 발표 초안의 `120FPS 이상 나오는 GPU로 변경해줘`는 현재 자산에서 초기 RTX 5060이 PUBG QHD 평균 130FPS를 이미 충족하므로 GPU 변경 시연이 되지 않는다. 또한 게임·해상도·평균 또는 1% low 기준이 빠지면 FPS 근거를 임의로 선택해야 한다.

따라서 발표 입력은 다음 문장으로 고정한다.

1. `중학교 3학년 조카가 처음 PC를 맞추려고 합니다. 배틀그라운드가 잘 돌아가는 컴퓨터를 원합니다.`
2. `배그 같은 게임이 잘 돌아갔으면 좋겠어요.`
3. `최소 견적으로 맞춰줘.`
4. `배그 QHD에서 평균 140FPS 이상 나오는 GPU로 바꿔줘.`
5. `이 변경안 적용해줘.`
6. `현재 견적의 호환 문제를 설명해줘.`
7. `호환되는 케이스로 바꿔줘.`

2026-07-16 최신 로컬 웹(`http://127.0.0.1:15174`)에서 위 7개 입력과 수동 케이스 교체, 견적 저장·비교·상세까지 같은 로그인 세션으로 다시 실행했다. 일반 변경의 확인 전 draft 변경은 없었고, 7번은 현재 `size` FAIL과 단일 검증 후보를 확인한 뒤 같은 요청에서 자동 반영됐다.

## 실제 웹 결과

전용 QA 계정으로 `http://127.0.0.1:15174`에서 다음 흐름을 확인했다.

1. 메인 AI 검색의 조카 상황 → 배그 용도 → 최소 견적 3턴 대화에서 최소 견적 카드 3개가 표시됐다.
2. 첫 카드를 적용한 셀프 견적은 8/8 부품, 1,634,590원, 종합 739/1000, PUBG QHD 평균 130FPS를 표시했다. 채팅도 같은 값으로 `완성 견적이 담겼습니다. 현재 종합 점수는 739점이며, 배그 QHD 예상 성능은 평균 130FPS입니다.`라고 안내했다.
3. 목표 FPS 요청은 RTX 5060 Ti 변경 미리보기 1건과 1,634,590원 → 1,740,100원 가격 차이를 표시했다.
4. `이 변경안 적용해줘` 후 GPU가 교체되고 종합 점수 736점, PUBG QHD 예상 성능 140FPS로 갱신됐다. 채팅도 같은 메시지를 갱신해 `요청한 변경이 반영되었습니다. 현재 종합 점수는 736점이며, 배그 QHD 예상 성능은 평균 140FPS입니다.`라고 안내했다.
5. Cooler Master NR200P V2를 의도적으로 선택하자 M-ATX 보드 장착 불가를 포함한 장착 실패 4건이 표시됐다.
6. `호환되는 케이스로 바꿔줘` 한 번으로 BE QUIET LIGHT BASE 900 DX 화이트 단일 검증안이 자동 적용됐다. 별도의 `적용해줘` 입력은 필요하지 않았다.
7. 적용 후 총액은 1,869,450원, 종합 점수는 738점, PUBG QHD 예상 성능은 140FPS로 갱신되고 호환 상태는 `이상 없음`이 됐다.
8. 가성비 견적과 고성능 견적을 각각 저장한 뒤 `/my/quotes`에서 두 견적의 예상가·종합 점수·부품 비교가 표시됐다.
9. 저장 견적 상세에서 구성 부품, 개별 가격, 호환성·전력·규격·성능·가격 검증 결과가 표시됐다.

## 종합 점수 해석 확인

GPU 교체 직후 PUBG QHD 평균 FPS는 130에서 140으로 올랐지만 종합 점수는 739에서 736으로 3점 내려갔다. 이는 표시 오류가 아니라 현재 1000점 정책이 게임 성능뿐 아니라 파워·장착·확장 여유를 함께 계산한 결과다.

| component | RTX 5060 | RTX 5060 Ti | 변화 |
|---|---:|---:|---:|
| 성능 | 213 | 230 | +17 |
| 호환·장착 안전성 | 220 | 220 | 0 |
| 병목·여유 균형 | 149 | 135 | -14 |
| 확장·운영 여유 | 87 | 81 | -6 |
| 근거 신뢰도 | 70 | 70 | 0 |
| 종합 | 739 | 736 | -3 |

RTX 5060 Ti 적용 후 파워 지속 부하 여유가 184W에서 149W로 줄고 정격 부하율이 69%에서 75%로 올라간 것이 주된 감점 원인이다. 따라서 발표에서는 `GPU 교체로 종합 점수가 상승했다`고 말하지 않고, `가격은 105,510원 증가하고 배그 QHD 평균은 130FPS에서 140FPS로 상승했으며 전체 Tool 검증은 통과했다`고 설명해야 한다. 종합 점수의 소폭 하락을 보여줄 경우에는 파워 여유까지 반영한 전체 구성 점수라는 점을 함께 말한다.

## AI 적용 후 안내 검증

- 추천 견적 카드 적용과 자연어 변경안 적용 모두 같은 처리 중 메시지에서 최종 성능 안내로 갱신됐다. 새 메시지가 중복 추가되지 않았다.
- 채팅의 점수는 `BuildGraph` 종합 점수, FPS는 화면과 동일한 `배그/QHD` 성능 조회 결과를 사용했다.
- 사용자가 성능 패널에서 다른 게임·해상도를 선택한 경우에는 적용 영수증 생성 시 그 선택을 고정하고 새 draft로 다시 조회한다. 이전 placeholder FPS는 사용하지 않는다.
- 성능 안내를 만들기 위한 추가 `/api/ai/build-chat` 호출은 없었으며, LLM이 점수나 FPS를 새로 생성하지 않았다.
- FPS 근거가 없는 경우에는 FPS 숫자를 표시하지 않고 점수만 안내하도록 자동 테스트로 고정했다.
- Tool FAIL 또는 종합 0점에서는 확인 가능한 FPS가 있어도 긍정적인 성능 안내를 숨기고 상단 경고 확인을 우선하도록 자동 테스트로 고정했다.
- 사용자별 적용 영수증은 현재 draft fingerprint가 일치할 때만 확정되며, 새로고침 후에도 같은 완료 안내를 다시 만들지 않는다.
- `현재 부품이 여유 있게 들어가는 케이스 추천해줘`는 현재 케이스를 제외하고 장착 여유가 실제 개선되며 전체 Tool 검증을 통과한 1위 후보 하나만 반환한다. 응답의 `AUTO_APPLY_VERIFIED_CASE_IMPROVEMENT` badge로 프론트가 적용 API를 정확히 한 번 호출하며, 일반 케이스 추천은 자동 적용하지 않도록 경계를 분리했다.
- Build Chat 답변 원문과 세션 저장값은 유지하면서 화면 표시만 문단화했다. 명시 줄바꿈을 보존하고 인라인 `1)·2)·3)` 후보는 항목별로 분리하며, 긴 설명은 최대 두 문장 단위 문단으로 표시한다. 새 답변의 문장 순차 노출과 모션 최소화 동작은 그대로 유지된다.
- 서버 검증 케이스 자동 적용 응답은 더 이상 적용 CTA가 있는 일반 미리보기 카드로 표시하지 않는다. 응답 당시 draft snapshot을 세션 로컬 영수증으로 보존해 `이전 케이스 → 변경 케이스`, `이전 예상가 → 변경 예상가`를 표시한다. 적용 후 같은 상태 메시지가 점수·FPS 결과로 갱신될 때도 새 reveal 대상으로 잡혀 세 문장이 순차 노출되는 것을 DOM mutation 기록으로 검증했다.

## 발표 품질 관찰 항목

- 최초 최소 구성은 blocking Tool `FAIL`이 0건이지만 성능 Tool `WARN` 때문에 하단 요약이 `호환 상태 주의 2개`로 표시된다. `모든 호환성 검사를 통과했다`는 설명은 `장착 불가 없이 핵심 호환 검증을 통과했다`로 표현하는 편이 화면과 정확히 맞는다.
- 저장 견적 상세와 일부 기존 설명에는 `내부 자산` 문구가 남아 있다. 기능 오류는 아니지만 사용자 발표 문구로는 `현재 판매 중인 부품` 또는 `확인된 부품 데이터`가 더 자연스럽다.
- 이번 브라우저 실행의 콘솔 오류 2건은 검증 도중 직접 시도한 교차 origin fetch의 CORS 차단이고, 1건은 이미 저장된 동일 견적의 중복 저장 `409`다. 제품 흐름에서 발생한 미처리 예외는 아니며 중복 저장은 화면에 `이미 저장되었습니다`로 처리됐다.

## 안전 불변식

- 첫 턴의 조카·`배틀그라운드` 문맥이 둘째·셋째 턴까지 유지된다.
- `최소 견적`은 숫자 예산 누락으로 다시 질문하지 않는다.
- 최소 견적 3안은 각각 8개 필수 카테고리를 가진다.
- blocking Tool FAIL 조합은 추천 카드에 포함하지 않는다.
- 목표 FPS 후보는 요청한 게임·해상도·지표의 FPS 근거가 있어야 한다.
- GPU 등 일반 변경은 먼저 미리보기를 만들고 사용자의 확인 후에만 적용한다. 케이스 `size` FAIL 상태의 서버 검증 단일 복구안만 명령형 요청과 함께 자동 적용한다.
- AI 적용 완료 안내의 점수와 FPS는 적용 후 현재 draft 및 화면 수치와 일치해야 한다.
- 수동 체크리스트 변경이나 미리보기 확인만으로는 AI 적용 완료 안내를 만들지 않는다.
- 교체 후 GPU와 케이스는 각각 1개만 존재한다.
- 호환 케이스 복구는 현재 케이스를 다시 추천하지 않는다.
- 저장 견적 비교와 상세는 저장 당시 snapshot을 사용한다.

## 발표 전 필수 준비

1. 공용 연습 계정이 아닌 발표 전용 계정을 사용한다.
2. 서버의 현재 quote draft와 AI 대화 기록을 초기화한다.
3. 비교용 고성능 저장 견적 1개를 미리 준비한다.
4. 최소 견적 3안과 첫 카드의 PUBG QHD 평균 FPS를 확인한다.
5. 현재 seed가 바뀌어 RTX 5060이 140FPS 이상이 되면 목표 FPS를 다시 정한다.
6. 의도적 실패용 케이스는 발표 직전 Tool 결과로 확인한다. 현재 seed에서는 Cooler Master NR200P V2가 M-ATX 보드와 장착 실패를 만든다.
7. `/api/health`, 메인 AI 검색, 셀프 견적, 내 견적함을 발표 직전에 한 번씩 연다.
