# 4분 발표 시나리오 실행 가이드

이 문서는 사용자 계정의 실제 쇼핑·AS 흐름을 4분 안에 재현하기 위한 발표용 절차다. 화면에 없는 기능을 말로 과장하지 않고, 현재 구현에서 확인 가능한 결과만 사용한다.

## 발표 전 준비

1. `docker compose up --build -d` 후 `GET /api/health`가 `UP`인지 확인한다.
2. 사용자 계정 `user@example.com`과 관리자 계정 `admin@example.com`으로 각각 로그인 가능한지 확인한다.
3. 사용자 계정에 진행 중인 AS 상담이 없어야 한다. 이전 연습 티켓의 상담방은 관리자가 종료한다.
4. AS 로그 fixture는 `apps/api/src/test/resources/as-rag-realistic-logs/remote_driver_display_tdr.jsonl`을 사용한다.
5. 사용자 화면과 관리자 티켓 화면을 별도 탭에 열어 둔다. 관리자는 새 티켓이 생성되면 즉시 처리할 준비를 한다.
6. GPU 하향 전 원래 GPU의 전체 상품명을 메모한다. 복구할 때 화면에 표시된 정확한 이름을 그대로 사용한다.

## 시간표와 통과 기준

| 시간 | 사용자 동작과 발표 문장 | 화면에서 확인할 결과 |
|---|---|---|
| 0:00~0:25 | `200만 원 예산으로 QHD 게임용 PC 추천해줘` 입력 | 175만~225만원 범위의 완성 견적 1~3개. 예산·게임 성능·호환성만 짧게 설명한다. |
| 0:25~1:05 | 가운데 추천안을 셀프 견적에 적용 | 8개 필수 카테고리와 Tool FAIL 없음. 부품별 설명은 생략한다. |
| 1:05~1:30 | `GPU를 한 단계 낮은 더 저렴한 제품으로 바꿔줘` 입력 후 미리보기 적용 | 가격 감소, 예상 게임 성능 감소, 호환성 통과가 한 화면에 보여야 한다. 이어 원래 GPU의 정확한 상품명에 `견적에 담아줘`를 붙여 복구한다. |
| 1:30~1:40 | `민수는 견적을 확정하고 조립 기사와 연결돼 PC를 배송받았습니다.`라고 설명 | 구매·기사·결제 화면은 열지 않는다. |
| 1:40~2:00 | `/support/new`에서 `게임 실행 중 화면이 멈추고 검은 화면이 나타났습니다.` 입력 | AS 접수 폼과 로그 선택 영역을 보여준다. |
| 2:00~2:35 | driver fixture 선택 후 로그 업로드·보관 동의 체크 | 추천 서비스가 원격지원 계열로 표시되고 Display 4101/드라이버 중단 근거가 요약된다. |
| 2:35~2:55 | AS 접수 후 티켓의 담당자 확인 자료 표시 | `DRIVER_ERROR_REPEAT`, Display/nvlddmkm 반복 신호, `REMOTE_POSSIBLE`을 확인한다. |
| 2:55~3:15 | 원격지원 사유를 확인하고 `원격지원 요청` 클릭 | 상태가 `REQUESTED`로 바뀐다. 사용자가 동의한 한 건의 원격지원 세션만 생성된다. |
| 3:15~3:45 | 관리자가 같은 티켓을 승인하고 Quick Assist 링크·드라이버 재설치 메모를 저장 | 사용자 화면이 5초 이내 갱신되어 `LINK_SENT`, 링크, 관리자 처리 내용이 나타난다. 완료 시 관리자가 티켓을 `RESOLVED`로 바꾼다. |
| 3:45~4:00 | `다짜조는 PC 추천에서 끝나지 않고, 구매 후 장애를 실제 데이터로 진단하고 원격 해결까지 연결합니다.` | 추천 → 변경 비교 → 로그 근거 진단 → 원격지원 연결 흐름을 요약한다. |

## AS 문구 정합성

- 현재 사용자 화면에는 AI가 별도 모달로 `시스템을 진단해도 될까요?`라고 묻는 기능이 없다.
- 발표에서는 `정확한 원인 확인을 위해 선택 구간 로그 전송에 동의합니다.`라고 설명하고 실제 로그 업로드·보관 동의 체크박스를 사용한다.
- 원격지원 전송 동의는 티켓 상세의 `원격지원 요청` 동작으로 보여준다.
- 원본 로그 전체나 개인정보는 화면에 노출하지 않는다. Display 이벤트 ID와 마스킹된 드라이버 반복 신호만 짧게 보여준다.

## 중단 조건

- 추천 견적이 175만~225만원 범위를 벗어나거나 필수 카테고리가 빠지면 발표를 진행하지 않는다.
- GPU 하향 미리보기에 Tool FAIL이 있거나 원래 GPU 복구 시 다른 상품이 선택되면 발표를 진행하지 않는다.
- AS 접수 전에 `진행 중인 AS 상담이 있습니다`가 보이면 기존 상담방을 먼저 종료한다.
- 원격지원 요청 1회로 활성 세션이 2건 생성되면 발표를 진행하지 않는다.
