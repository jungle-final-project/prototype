# 셀프견적 리디자인 — 멘토 피드백 반영 계획 (2026-07-06)

> 근거: 멘토 피드백 전문(팀 공유). 핵심 지적 — "사용자가 처음 봤을 때 무엇을 사야 하는지,
> 무엇부터 해야 하는지를 UI가 알려줘야 한다. 지금은 기능을 보여주는 화면이지, 길을 잃지 않게
> 만드는 화면이 아니다." 셀프견적 화면은 전권 위임받음.

## 취사선택 결정

| 쟁점 | 결정 | 근거 |
|---|---|---|
| 그래픽 방향 A(실감) vs B(정보형) | **B — 정보형 UI, 그래프는 보조** | 멘토도 B가 현실적이라 판단. A(실사/조립 애니메이션)는 P2 |
| AI-first vs Self-quote-first | **이 화면은 Self-quote-first + AI CTA 상시 노출** | 챗봇은 전역 사이드바로 이미 존재. 빈 견적 첫 화면에 "AI로 시작" 배너를 크게 노출해 두 흐름 모두 명시 |
| 범위 | **P0 전부 + P1 중 순차 가이드·후보 회색 처리** | 견적 비교(버전 A/B/C)·기존 PC 업그레이드 흐름은 R1로 |

## R0 (이번 PR) — 구현 항목

1. **견적 체크리스트(품목 지도)** — 좌측 컬럼. 권장 순서(CPU→메인보드→RAM→GPU→SSD→파워→케이스→쿨러)로
   8품목을 번호와 함께 나열: 완료(✓ 부품명·가격) / 비어 있음(○). 클릭 = 해당 후보 패널 열기.
   진행 N/8과 **총액 상시 표시**.
2. **순차 선택 가이드** — "다음: 메인보드를 선택해 주세요" 배너 + 체크리스트·보드에서 다음 슬롯 강조(펄스 링).
   전문가는 아무 슬롯이나 바로 클릭 가능(가이드는 안내지 강제 아님).
3. **AI 시작 CTA** — 견적이 비어 있으면 상단에 "뭘 골라야 할지 모르겠다면 — AI에게 예산·용도만 말해 주세요"
   배너 + [AI로 시작] 버튼(챗봇 열기+프리필) + [직접 고르기] 버튼(CPU 패널 열기).
4. **보드 보조화** — topology 배경 이미지·PCIe/24핀 장식 라벨 제거. 슬롯 카드는 제품 이미지·이름·가격·상태
   중심(장착 시 실제 상품 이미지 — 기존 동작 유지·강조). 그래프는 "호환성 이해 도구"로만.
5. **호환 색상 통일** — 정상 연결 = **초록**(기존 teal→green), 주의 = 노랑, 불가 = 빨강(더 굵게), 미검증 = 회색.
   장착 카드 뱃지도 동일 체계(정상 = 초록 "호환 가능").
6. **후보 패널: FAIL 숨김 → 회색 표시 + 사유** — 모든 후보를 보여주되 비호환은 비활성(회색)
   + "선택 불가 · {사유}" 표기. 서버는 이미 status+사유를 내려주므로 프론트만 변경.
   전부-FAIL 페이지 자동 스킵 로직 제거(더 이상 불필요).
7. **담기/교체 문구** — 빈 슬롯 = "담기", 채워진 단일 슬롯 = "교체", 교체 대상 지정 시 = "이걸로 교체"
   (기존 로직 유지·문구 점검).
8. **사용자 언어** — 엣지/뱃지 라벨에서 전문용어 제거 점검(대부분 한글화돼 있음 — "호환 가능/주의/장착 불가" 통일).

## R1 (후속)

- 견적 비교(가성비/균형/고성능 3안 나란히, 변경 부품 diff)
- 기존 구매 견적 불러오기 → "요즘 게임이 잘 안 돌아가요" → AI가 교체 1~2개 제안(최소/고성능 안)
- AI 수정 제안이 영향 주는 부품 강조("가격 낮춰줘" → 바뀌는 슬롯 깜빡임)
- 강제 선택(호환 무시하고 교체) + 영향 부품 빨간 표시

## P2 (장기)

- 2D/3D 조립 시각화(실사 이미지·장착 애니메이션)
- AS Agent형 진단 흐름(별도 트랙 — PC Agent 보류 해제 후)
